# ImageGuard WebUI 审核控制台设计

## 背景

当前页面把审核记录和配置放在两个 Tab 中。记录页能看到历史、供应商调用统计和审核缓存，配置页能修改规则、模型、范围、处罚和图片处理参数，但信息层级较平，配置字段较多，移动端扫描成本较高。

本次目标是把页面优化为一个统一的“审核控制台”，同时改善日常巡检和配置维护，不改变插件后端行为。

## 目标

- 让用户进入页面后先看到审核状态、风险数量和近期处理结果。
- 让用户可以从同一套导航快速进入完整记录和配置分组。
- 保留现有 `AstrBotPluginPage` Bridge、Web API、字段 ID 和保存语义。
- 让表格、配置表单、弹窗和图片预览支持键盘与移动端使用。
- 在浅色和 AstrBot 深色主题下保持清晰的状态色和足够对比度。

## 非目标

- 不增加新的后端 API 或持久化字段。
- 不实现供应商管理器；供应商增删排序仍由 AstrBot 插件配置页负责。
- 不改变审核缓存、处罚、上报或 LLM 调用逻辑。
- 不引入前端构建工具、第三方运行时依赖或新的服务端页面。

## 方案

### 页面外壳

将现有单页改为以下结构：

```text
┌─────────────────────────────────────────────────────────┐
│ ImageGuard / 审核控制台       数据状态  主题  刷新      │
├───────────────┬─────────────────────────────────────────┤
│ 工作区         │ 审核概览                                  │
│  审核概览      │ 记录数  违规数  撤回成功率  缓存节省调用     │
│  审核记录      │ ┌───────────────────────┬───────────────┐ │
│  配置中心      │ │ 最近审核记录           │ 供应商调用     │ │
│                │ │                       │ 审核缓存       │ │
│ 配置目录       │ └───────────────────────┴───────────────┘ │
│  审查规则      │ 配置分组预览 / 快速定位                     │
│  LLM 供应商    │                                             │
│  范围与处罚    │                                             │
│  图片与调试    │                                             │
└───────────────┴─────────────────────────────────────────┘
```

桌面端使用左侧导航和主工作区；移动端将导航折叠为横向滚动条，主内容改为单列。

### 视觉语言

- `--ink`：深墨色，用于主文字和主导航，表达稳定的控制台基底。
- `--mint`：墨绿色，用于正常状态、成功操作和主行动。
- `--coral`：珊瑚红，只用于违规、失败和危险操作。
- `--amber`：琥珀色，用于待检查或提醒状态。
- `--paper` / `--surface`：浅灰绿工作底色和白色内容面。
- 数字使用 `font-variant-numeric: tabular-nums`，便于比较。
- 用窄色标连接导航、摘要和状态标签，作为页面的唯一强记忆点；不使用大面积渐变或装饰性背景。

### 审核概览

概览使用 `audit/list` 的单次响应派生数据：

- 历史记录：`records.length`。
- 已记录违规：当前历史记录数，文案明确为历史记录而非实时检测量。
- 撤回成功率：由已有 `recalled` 字段计算；无记录时显示 `—`。
- 缓存节省调用：来自 `cache_stats.images_skip_audit`。
- 审核表：保留时间、用户、来源、理由、图片、撤回、禁言和删除操作。
- 侧栏：供应商调用统计、缓存统计、配置状态；配置状态只反映本页可验证的配置存在性，不声称外部服务健康。

完整记录页继续使用分页，每页 15 条。空状态、加载状态和失败状态均占据表格完整宽度，并提供明确的下一步操作。

### 配置中心

配置中心保留现有表单元素 ID 和字段转换逻辑，增加可定位的分组：

1. 审查规则：敏感文字、违规画面描述、自定义识图引导词。
2. LLM 供应商：供应商摘要、推理强度、最大 Token、超时。
3. 范围与处罚：群/私聊范围、抽查概率、撤回、禁言、上报目标。
4. 图片与调试：压缩上限、历史条数、临时图片、LLM 调试日志。

分组目录使用 hash 保存当前视图，例如 `#overview`、`#records`、`#config-rules`。目录点击只改变页面定位，不新增服务器路由。

### 交互和错误处理

- 刷新复用 `audit/list`，请求期间展示结构化占位并防止重复刷新。
- 删除单条记录和清空历史继续使用确认弹窗。
- 保存配置前提示“保存后会重载插件”，请求期间按钮进入忙碌状态，成功和失败消息放入 `aria-live="polite"` 状态区。
- 异步失败消息包含失败原因和重新加载或重试动作。
- 图片预览支持 `Escape` 关闭、点击遮罩关闭、可见焦点和原始触发按钮焦点恢复。
- 所有图标按钮提供 `aria-label`；装饰性图标不作为唯一语义来源。
- 所有表单控件有可点击标签、正确的输入类型和可见焦点；不拦截粘贴。
- 保留 `prefers-reduced-motion`，动画只使用透明度或变换。

## 数据流

```text
AstrBotPluginPage.ready()
        │
        ├── apiGet("audit/list")
        │      ├── 概览指标
        │      ├── 审核记录分页
        │      ├── 供应商统计
        │      └── 缓存统计
        │
        └── apiGet("audit/config")  ── 配置中心
                                      │
                                      └── apiPost("audit/config/update")
```

页面不直接访问插件 KV，不缓存 API Key，也不复制或重写供应商配置。

## 验证计划

- 使用 `python -m json.tool _conf_schema.json` 校验配置 schema。
- 使用 `python -m py_compile main.py cache.py image_processor.py` 校验插件 Python 文件。
- 使用 Node 对页面脚本进行语法检查，并静态确认所有现有 API 路径仍存在。
- 在桌面宽度验证导航、指标、表格和配置布局。
- 在窄屏验证导航滚动、记录信息完整性、弹窗和输入控件可用性。
- 使用 Web Interface Guidelines 复查语义 HTML、标签、焦点、异步状态、图片尺寸、深色模式、动效和空状态。

## 变更边界
